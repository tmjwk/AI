🏗️ ARCHITEKTURY I WZORCE

Agent Map-Reduce

Orchestrator (conductor) rozdaje zadania sub-agents, ci wysyłają wyniki. Jak Hadoop/Spark dla agentów. Znany też jako: LLM MapReduce, Agentic MapReduce, Fan-out/fan-in, Agentic wide search.

3-Layer Agent Architecture

1. Core runtime (capability-agnostic, MCP, lazy loading)
2. Shared plugins (skills, MCP servers, hooks)
3. Per-client config (thin, bez forków)

Agentic OS — 4 poziomy

1. Skills & Automation (workflow audit)
2. Memory & State (Obsidian, structured map)
3. Interface (visual wrapper/dashboard)
4. Distribution (packaging dla teams/clients)

Brain in the Middle

Klasyczna app: input → custom software → output.
Agent app: input → AI agent (skills, MCP) → output. Agent zastępuje całe custom software.

💡 Fable Brain Prompting — 8 instrukcji od Anthropic

1. Act, do not overplan — commit to recommendation
2. Lead with the outcome — bottom line first
3. Ground every claim — audit against evidence
4. Stop only at real boundaries
5. Assess, do not act uninvited
6. Give the reason
7. Provide the context
8. Match effort to the task

Limit: To działa dla komunikacji, ale nie tworzy judgment. Model może pewnie podać złą odpowiedź. Nie daje agency (brak tools/internet).

🔧 Harness Engineering — x6 różnica

Ten sam model może dać 6x lepsze wyniki zależnie od harness. Kluczowe: context management, memory as hint, tool validation, feedback loops.

⏰ Custom Scheduling dla Digital Employee

Custom "mycron" zamiast system crontab. Browser session persistence, hot-reload (60s), RAND_DELAY (anti-thundering herd), per-minute dedup, isolated per-user logging.
